# PR: test: add deterministic output tests for reporting trend analysis

## Summary

Closes #312. Adds 21 focused tests that verify `get_trend_analysis` and the
new `get_trend_analysis_multi` function produce **identical, deterministic
output for identical historical inputs**, regardless of call order, ledger
timestamp, or user address.

## Changes

### `reporting/src/lib.rs`
- **Fix**: removed duplicate `PolicyPage` struct definition that caused
  `E0428` compile errors on fresh clones.
- **New**: `get_trend_analysis_multi(env, user, history: Vec<(u64, i128)>)`  
  Accepts an ordered sequence of `(period_key, amount)` pairs and returns
  `Vec<TrendData>` (one entry per adjacent pair). Returns empty Vec for fewer
  than two points.

### `reporting/src/tests.rs`
- **Fix**: added `use super::*` and correct testutils trait imports
  (`Address as _`, `Ledger`, `LedgerInfo`, `storage::Instance as _`) so
  tests compile against Soroban SDK 21.7.7.
- **Fix**: restored `create_test_env()` helper removed in a prior refactor.
- **New**: 21 deterministic trend analysis tests covering:

| Category | Tests |
|---|---|
| Repeated calls | `test_trend_deterministic_repeated_calls` |
| User independence | `test_trend_deterministic_different_users_same_amounts` |
| Timestamp independence | `test_trend_deterministic_across_timestamps` |
| Scalar edge cases | zero/both-zero/no-change/full-decrease/exact-double/large values |
| Multi sparse (2 pts) | `test_trend_multi_sparse_two_points` |
| Multi dense (5 pts) | `test_trend_multi_dense_five_points` |
| Multi determinism | `test_trend_multi_dense_deterministic` |
| Boundary: 0 or 1 pt | `test_trend_multi_empty_returns_empty`, `test_trend_multi_single_point_returns_empty` |
| Boundary windows | zero crossings, flat window, alternating up/down |
| User independence | `test_trend_multi_deterministic_across_users` |
| Timestamp independence | `test_trend_multi_deterministic_across_timestamps` |

### `reporting/README.md`
- Added full public API reference, trend analysis determinism guarantee, test
  coverage summary, and security notes.

### Workspace `Cargo.toml` files
- Pinned `soroban-sdk` to `=21.7.7` across all workspace crates to eliminate
  transitive version conflicts on fresh clones.

## Security notes

- Trend functions are **pure** — no storage writes, no auth required, no
  side-effects. This eliminates any attack surface introduced by this PR.
- All arithmetic uses `i128`; overflow protection is enforced at the profile
  level (`overflow-checks = true`).
- No new admin surfaces or privileged paths were introduced.
